Rendszerarchitektúra és 0 Ft-os működési modell

A Tüskevár DSE honlap célarchitektúrája saját szerver nélküli, statikus publikálásra épül. A havi üzemeltetési költség célértéke 0 Ft, a domain regisztrációs díján kívül.

1. Szolgáltatások szerepe

  • ZTS / ZeroTime Services: a tuskevardse.hu domain regisztrátora.
  • Cloudflare: DNS kezelés, HTTPS tanúsítvány, weboldal publikálás, custom domain.
  • GitHub: forráskód tárolás, verziózás, jogosultságkezelés, push alapú deploy indítás.
  • Codex: lokális fejlesztés, tartalmi és kódbeli módosítások, tesztelés előkészítése.
  • npm / Node.js: lokális fejlesztői parancsok futtatása és production build készítése.
  • Google Workspace for Nonprofits: saját domaines email és nonprofit irodai eszközök.

2. Miért statikus weboldal?

A projekt Next.js alapú, de statikus exportként működik. Ez azt jelenti, hogy a build során minden publikálható oldal HTML/CSS/JavaScript állományként készül el az out mappába.

A Next.js dokumentáció szerint az output: 'export' beállítással a next build HTML fájlt generál az útvonalakhoz, és az így készült statikus assetek bármilyen statikus webkiszolgálón hosztolhatók. Forrás: Next.js Static Exports.

3. A projekt fontos technikai beállításai

A next.config.ts célbeállításai:

output: "export"
trailingSlash: true
images: {
  unoptimized: true
}

Ezek azért fontosak, mert a Cloudflare oldalon nem akarunk saját Node.js szervert vagy fizetős backend runtime-ot.

4. Működési lánc

  1. A fejlesztés lokálisan történik Codex segítségével.
  2. A fejlesztő gépen fut az npm és a Next.js dev server.
  3. A módosításokat lokálisan ellenőrizni kell: lint és build.
  4. A kész változás commitba kerül.
  5. A commit GitHubra pusholódik.
  6. Cloudflare a GitHub push alapján automatikusan buildel és deployol.
  7. A látogatók a https://tuskevardse.hu címen érik el az oldalt.

5. 0 Ft-os költségkontroll

A havi 0 Ft-os működés kulcsa: nincs saját szerver, nincs fizetős adatbázis, nincs fizetős API, nincs fizetős emailküldő szolgáltatás, nincs kötelező fizetős CI runner.

Kerülendő, ha a 0 Ft garantált cél:

  • fizetős OpenAI vagy más AI API kulcs bekötése publikus funkcióhoz,
  • serverless adatbázis kötelező runtime-ként,
  • Cloudflare KV, D1 vagy R2 kötelező működési elemként, ha fizetős limitbe csúszhat,
  • külső emailküldő API,
  • saját VPS vagy webhosting,
  • fizetős GitHub Actions runner igény.

6. Kritikus pontok

  • A domain DNS kezelése Cloudflare-ben legyen, különben a Worker custom domain nehezen kezelhető.
  • Az email DNS rekordokat nem szabad proxied módba tenni.
  • A build parancs mindig fusson le lokálisan push előtt.
  • Publikus repositoryba token, jelszó, API kulcs nem kerülhet.
  • Ha Google Workspace-re vált a levelezés, az MX/SPF/DKIM/DMARC rekordokat kontrolláltan kell cserélni.
