Lokális tesztelés és produkcióba vitel

Ez a dokumentum azt írja le, hol fut az npm, hogyan kell lokálisan tesztelni, és hogyan kerül a módosítás automatikusan produkcióba GitHub és Cloudflare segítségével.

1. Hol fut az npm?

Fejlesztés közben az npm a lokális gépen fut, a C:\Projects\Tuskevar-DSE mappában. Cloudflare deploy közben az npm a Cloudflare build környezetében fut. A GitHubon önmagában nem fut npm, hacsak nincs külön GitHub Actions workflow beállítva.

2. Első lokális indítás

cd C:\Projects\Tuskevar-DSE
npm ci
npm run dev

A fejlesztői oldal tipikusan itt érhető el:

http://127.0.0.1:3000/
http://localhost:3000/

3. npm indítás és leállítás

Indítás:

npm run dev

Leállítás ugyanabban a terminálban:

Ctrl + C

Ha rákérdez:

Terminate batch job (Y/N)? Y

4. Kötelező ellenőrzések push előtt

npm run lint
npm run build

A npm run build sikeres futása után a Next.js statikus exportot készít az out mappába. Ez a Cloudflare által publikálandó tartalom.

5. Cloudflare build/deploy beállítás

A projekt jelenlegi Cloudflare Worker/static asset jellegű beállításához:

Build command:
npm run build

Deploy command:
npx wrangler deploy --assets=./out --name=tuskevardse --compatibility-date=2026-06-16

Ha Cloudflare Pages tiszta static hosting módban van használva, akkor jellemzően csak build command és output directory kell: 

Build command:
npm run build

Build output directory:
out
A projektben korábban Worker alapú Git deploy felület jelent meg, ezért ott a wrangler deploy --assets=./out parancs volt szükséges. Ha később tiszta Pages projektként újra létrejön, akkor a wrangler deploy parancs nem feltétlenül kell.

6. Automatikus produkcióba vitel

  1. Lokális fejlesztés és tesztelés elkészül.
  2. git commit készül.
  3. git push feltölti a változást GitHubra.
  4. Cloudflare érzékeli a GitHub push-t.
  5. Cloudflare lefuttatja a build parancsot.
  6. Cloudflare publikálja az új verziót.

Kézi Cloudflare deploy csak akkor kell, ha:

  • hibás beállítással futott le egy korábbi deploy,
  • Cloudflare build/deploy parancsot módosítottatok,
  • ugyanazt a commitot akarjátok újrafuttatni.

7. Ellenőrzési pontok deploy után

  • Cloudflare Deployments: build sikeres-e.
  • Preview URL: betölt-e az oldal.
  • Production domain: https://tuskevardse.hu.
  • www.tuskevardse.hu működik-e vagy átirányít-e.
  • Kapcsolat gombok, email linkek, navigáció, mobil nézet.

8. Gyakori hibák

  • Deploy command required: Worker alapú Cloudflare projektben meg kell adni a deploy parancsot.
  • compatibility_date is required: wrangler deployhoz kell --compatibility-date.
  • out mappa hiányzik: a npm run build nem futott le, vagy nem static export készül.
  • Régi tartalom látszik: cache, hibás domain rekord, vagy nem a latest commit van deployolva.
  • Build lokálisan jó, Cloudflare-en nem: Node/npm verzió vagy Cloudflare parancs eltérés lehet.
