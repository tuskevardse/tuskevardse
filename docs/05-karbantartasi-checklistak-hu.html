Karbantartási checklisták

Ezek a listák a későbbi tartalmi módosításokhoz, technikai frissítésekhez, domain/email ellenőrzésekhez és hibakereséshez adnak gyakorlati sorrendet.

1. Tartalmi módosítás checklist

  1. Meg kell határozni, melyik oldal vagy szekció változik.
  2. A szöveg látogatóknak szóljon, ne belső projektleírás legyen.
  3. Kerülendő: “felület”, “koncepció”, “AI asszisztens”, “később bővíthető”, ha nem látogatói érték.
  4. Codex módosítja a fájlokat.
  5. Lokális ellenőrzés: npm run lint.
  6. Lokális build: npm run build.
  7. Commit és push.
  8. Cloudflare deploy ellenőrzése.

2. Technikai módosítás checklist

  1. Először lokálisan kell módosítani.
  2. Meg kell nézni, hogy érinti-e a statikus exportot.
  3. Nem kerülhet be szerveroldali kötelező funkció, ha a 0 Ft-os működés cél.
  4. Nem kerülhet be titkos kulcs publikus kódba.
  5. npm run lint és npm run build kötelező.
  6. Ha build konfiguráció változik, Cloudflare beállítást is ellenőrizni kell.

3. Domain ellenőrzés checklist

  • ZTS-ben a névszerverek a Cloudflare által adott két nameserverre mutatnak-e.
  • Cloudflare-ben a zone aktív-e.
  • tuskevardse.hu és www.tuskevardse.hu custom domainként fel van-e véve.
  • HTTPS tanúsítvány aktív-e.
  • A régi 77.111.95.164 IP nem irányítja-e el a fő webes forgalmat.

4. Email ellenőrzés checklist

  • MX rekord DNS only.
  • Mail/smtp/pop rekordok DNS only.
  • SPF rekord csak az aktuális email szolgáltatóra mutasson.
  • Google Workspace esetén SPF: v=spf1 include:_spf.google.com ~all.
  • DKIM rekord felvéve a Google Admin Console alapján.
  • DMARC rekord létezik, induláskor lehet p=none.

5. Rollback lehetőségek

Ha egy új deploy hibás:

  1. Cloudflare Deployments alatt vissza lehet állni korábbi sikeres deployra, ha a felület kínál rollback opciót.
  2. Gitben vissza lehet vonni a hibás commitot új revert committal.
  3. Nem javasolt lokálisan git reset --hard használata, ha nem teljesen világos, milyen változásokat törölne.
git revert <hibas-commit-sha>
git push

6. Havi/alkalmi karbantartás

  • Cloudflare deploy státuszok ellenőrzése.
  • GitHub repository hozzáférések ellenőrzése.
  • Google Workspace felhasználók és jogosultságok ellenőrzése.
  • Domain lejárati dátum ellenőrzése ZTS-nél.
  • Kapcsolati email és űrlap tesztelése.
  • Mobil nézet gyors ellenőrzése.

7. Kritikus szabályok

A produkcióba kerülés egyetlen normál útja: lokális teszt → commit → push → Cloudflare automatikus deploy. Cloudflare-ben kézzel csak beállítást vagy hibás deploy újrafuttatását kell kezelni.
  • Nem módosítunk produkciós tartalmat közvetlenül Cloudflare-ben.
  • Nem töltünk fel kézzel módosított buildet a GitHub megkerülésével.
  • Minden tartós változásnak Git commitban kell szerepelnie.
  • DNS módosítás előtt fel kell írni a régi rekordokat.
  • Email DNS váltást csak kontrollált időpontban szabad végezni.
