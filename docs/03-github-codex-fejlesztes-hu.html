GitHub és Codex fejlesztési folyamat

Ez a dokumentum a forráskód kezelését, a GitHub jogosultságokat, a Codex alapú fejlesztést és a commit/push alapú publikálási rendet írja le.

1. Repository

A projekt GitHub repositoryja:

https://github.com/tuskevardse/tuskevardse

A lokális projektmappa:

C:\Projects\Tuskevar-DSE

Az alapértelmezett branch:

main

2. Jogosultságok

Pusholni csak olyan GitHub felhasználó tud, akinek a repositoryhoz legalább write jogosultsága van. Ha 403 Permission denied hiba jelenik meg, akkor általában nem a token rossz, hanem az aktuálisan bejelentkezett GitHub felhasználónak nincs írási joga.

Ellenőrizendő:

  • GitHub repository → Settings → Collaborators and teams.
  • A felhasználó szerepel-e a hozzáférők között.
  • Elfogadta-e az emailes meghívót.
  • Van-e write vagy admin jogosultsága.

3. Hitelesítés CLI-ből

Windows alatt a Git Credential Manager használata javasolt:

git config --global credential.helper manager

Ha a GitHub felhasználónév és token egyszer helyesen megadásra kerül, a credential manager eltárolja, így nem kell minden pushnál újra megadni.

Publikus repositoryba token, jelszó, Cloudflare API token, Google verification secret vagy bármilyen érzékeny adat nem kerülhet.

4. Codex szerepe

A Codex a lokális projektmappában dolgozik. Tipikus feladatai:

  • tartalmi szövegek módosítása,
  • Next.js komponensek és oldalak módosítása,
  • lokális tesztek futtatása,
  • build hibák javítása,
  • dokumentáció készítése,
  • commit és push előkészítése.

A fejlesztés lokálisan történik, nem a Cloudflare-en és nem a GitHubon. A GitHub a verziózott forráskód központi helye, a Cloudflare pedig a publikált oldal kiszolgálója.

5. Napi fejlesztési folyamat

  1. Codex módosítja a szükséges fájlokat.
  2. Lokális ellenőrzés: npm run lint.
  3. Lokális build: npm run build.
  4. Változások áttekintése: git status és szükség esetén git diff.
  5. Stage: git add ..
  6. Commit: git commit -m "Rövid leírás".
  7. Push: git push.
  8. Cloudflare automatikusan buildel és publikál.

6. Ajánlott parancsok

git status
git diff
npm run lint
npm run build
git add .
git commit -m "Update site content"
git push

7. Mikor kell új branch?

Kisebb tartalmi javítások mehetnek közvetlenül main ágra, ha ez a csapat elfogadott munkarendje.

Új branch javasolt:

  • nagyobb redesign,
  • navigációs struktúra módosítása,
  • űrlap vagy Google integráció átalakítása,
  • Cloudflare vagy build konfiguráció változtatása.

8. GitHub és Cloudflare kapcsolat

Cloudflare a GitHub repositoryhoz kapcsolódik. Ha a main ágra push történik, Cloudflare új deployt indít. Kézi Cloudflare deploy csak hibás korábbi futás vagy beállításváltozás után szükséges.
